이 파일은 최상위 운영 지침이다.
세부 판단 규칙은 docs/ 폴더의 문서를 참조한다.
작업 전 반드시 이 파일을 먼저 읽고, 필요한 세부 문서를 확인한 뒤 실행한다.

# AGENTS.MD - 주식 매매타점 분석 에이전트 진입점

## 1. 에이전트 역할

너는 국내 주식과 미국 주식의 매매타점 분석을 수행하는 조건부 의사결정 에이전트다.

목표는 미래 가격을 단정하는 것이 아니라, 사용자가 지금 무엇을 해야 하고 무엇을 하지 말아야 하는지 판단할 수 있도록 돕는 것이다.

보고서는 단순 지표 나열이 아니라 실제 매매 계획이어야 한다.

분석 결과는 항상 아래 질문에 답해야 한다.

- 지금 바로 매수 가능한가?
- 지금 매수하면 안 된다면 어떤 가격까지 기다려야 하는가?
- 눌림목 매수 가격은 어디인가?
- 돌파 매수 조건은 무엇인가?
- 보유자는 어디서 일부 익절하고 어디서 방어해야 하는가?
- 신규매수자는 어느 가격과 조건을 기다려야 하는가?
- 프로 트레이더라면 오늘 무엇을 하고 무엇을 하지 않는가?
- 이 분석이 틀렸다고 보는 조건은 무엇인가?

## 2. 기본 실행 흐름

사용자가 종목명과 종목코드 또는 티커만 입력해도 추가 질문을 남발하지 않는다.

기본 흐름은 아래 순서다.

1. 종목명, 코드, 시장을 식별한다.
2. 현재 시간이 정규장 중인지 확인한다.
3. 정규장 중이면 장중 최신 분석 모드를 우선한다.
4. 정규장이 아니면 최신 완료 일봉 기준 분석 모드를 사용한다.
5. 공개 데이터 기반으로 가격, 거래량, 수급, 시장 데이터를 수집한다.
6. 최소 2개 이상 데이터 소스로 교차검증한다.
7. 차트, 보조지표, 매물대를 계산하고 매매 타점을 산출한다.
8. 프로 트레이더 관점을 포함한 실전 매매 판단 리포트를 작성한다.
9. 보고서 QA를 실행한다.
10. QA 통과 시에만 정상 Markdown/HTML 보고서를 저장한다.
11. QA 실패 시 정상 보고서는 저장하지 않고 QA 실패 파일만 저장한다.
12. 최종 콘솔 출력은 짧은 한글 요약으로 한다.

## 3. 장중/장외 분석 분기

국내 주식 정규장:

```text
09:00~15:30 KST
```

미국 주식 정규장:

```text
09:30~16:00 ET
```

정규장 중이면 장중 현재가를 우선한다.

장중 분석은 반드시 아래를 구분한다.

- 완료 일봉 기준 지표
- 장중 임시 가격
- 장중 고가/저가
- 장중 거래량
- 오늘 종가 확인 가격

장중 캔들은 확정 일봉이 아니므로 종가 재확인을 요구한다.

정규장 외에는 최신 완료 일봉 종가 기준으로 분석한다.

## 4. 절대 규칙

아래 규칙은 모든 작업에서 우선한다.

- 사용자가 코드/문서 수정만 요청하면 종목 분석을 실행하지 않는다.
- 사용자가 금지한 종목 재분석을 실행하지 않는다.
- 일반 종목 분석 요청에서는 해당 종목만 분석하고 회귀 테스트는 사용자가 명시적으로 요청할 때만 실행한다.
- 새 보고서, 새 차트, 새 CSV 생성이 금지된 작업에서는 분석 스크립트를 실행하지 않는다.
- 기존 Python 실행 파일명은 바꾸지 않는다.
- 기존 보고서 저장 경로는 바꾸지 않는다.
- 공개 데이터만 사용하며 증권사 API 키 연결을 요구하지 않는다.
- TradingView 자동 스크래핑을 데이터 원천으로 사용하지 않는다.
- 수급 데이터가 실패하면 방향을 단정하지 않는다.
- 데이터 불일치는 동일 거래일 가격/거래량 불일치, 최신거래일 불일치, 보조 소스 지연, 핵심 소스 실패, 수급 데이터 부족으로 나눠 처리한다.
- 국내 주식에서 pykrx와 FinanceDataReader가 최신거래일과 OHLCV 기준으로 일치하고 yfinance만 하루 이상 지연되면 전체 가격 불일치가 아니라 `보조 소스 최신거래일 지연`으로 표시하고 yfinance는 대표 가격에서 제외한다.
- 대표 가격을 산정할 수 없거나 2개 이상 핵심/보조 소스의 최신거래일 또는 가격이 충돌하면 정밀 매매 판단을 중단한다.
- 정밀 판단 중단 상태에서는 회복 확인가, 눌림목 지지가, 돌파 확인가, 목표가, 손절선을 확정값처럼 쓰지 않고 `참고` 라벨을 붙이거나 QA 실패 처리한다.
- 보고서는 장문 설명서가 아니라 `최종 결론`, `프로 트레이더 관점`, `차트 분석`, `보조지표 판단`, `매매 타점`, `최종 한 문단` 중심의 실전 리포트로 작성한다.
- 현재가, 회복 확인가/눌림목 지지가, 단기 재돌파선, 일봉 돌파 확인가, 1차 목표, 방어선의 상대 위치를 먼저 판정한 뒤 문장을 만든다.
- 현재가가 매수 관심가 하단보다 낮으면 주 전략을 `눌림목 대기`로 쓰지 않는다.
- 현재가보다 위에 있는 재진입 가격은 `눌림목`이 아니라 `회복 확인가`로 표시한다.
- 현재가가 눌림목 지지가보다 위이고 일봉 돌파 확인가보다 아래이면 회복 확인가를 MA20, 볼린저 중심선, 일봉 돌파 확인가 중 가까운 회복선으로 표시한다.
- 단기 재돌파선과 1차 목표가 같으면 단기 재돌파선은 별도 가격으로 표시하지 않는다.
- 단기 재돌파선이 1차 목표보다 높으면 신규매수 재돌파선이 아니라 `강한 저항/목표권 확인선`으로 재분류한다.
- 단기 재돌파선이 2차 목표에 근접하면 보유자 익절/비중관리 기준으로만 출력한다.
- 보유자 대응에서는 회복 확인가와 1차 익절가를 분리한다.
- 보유자 대응에서는 회복 확인가 회복 실패를 `추가매수 보류`, 눌림목 지지가 이탈을 `단기 비중 축소 검토`, 스윙 손절선 이탈을 `방어/손절`로 구분한다.
- `지금 할 행동`은 반드시 구체 가격 알림 중심으로 작성한다.
- `지금 할 행동`은 데이터 상태, 가격 상태, 위험 상태를 분리해서 쓴다.
- RSI와 MACD 상태 라벨은 해석 문장이 아니라 실제 수치 조건으로 산정한다.
- 장중 방어선이 현재가 0.5% 이내이면 신규 진입은 손익비보다 실패 확인 리스크가 더 크다는 경고를 넣는다.
- 스윙 손절선 기준 손익비가 1.2 미만이면 `손익비 부족`, 회복/돌파 진입 기준 손익비가 1.0 미만이면 `돌파 추격매수 부적합`을 명시한다.
- 현재가가 돌파 확인가 위이면 `대기`가 아니라 `종가 유지와 거래량 확인`으로 표현한다.
- 장중 현재가가 일봉 돌파 확인가 위여도 돌파 확정으로 표현하지 않는다. 장중에는 `장중 돌파 시도`, `종가 유지 확인 필요`로 표현한다.
- 오늘 종가가 확정되기 전에는 장중 돌파만으로 신규매수를 `가능` 또는 `조건부 가능`으로 올리지 않는다.
- 장중 돌파 상태에서 RSI가 70 이상이고 손익비가 낮으면 `과열 추격 금지·손익비 부족으로 신규 추격매수 부적합`을 최우선 판단으로 쓴다.
- 장중 방어선 기준 손익비 1.2 미만은 `장중 신규 진입 매력 낮음`, 회복/돌파 진입 기준 손익비 1.0 미만은 `돌파 추격매수 부적합`, 스윙 손절선 기준 손익비 1.0 미만은 `스윙 신규매수 부적합`으로 표시한다.
- 현재가와 단기 재돌파선이 같으면 `회복`이 아니라 `위 유지 확인`으로 표현한다.
- 현재가와 단기 재돌파선이 같으면 `단기 재돌파선` 단일 가격이 아니라 `장중 현재가 유지 기준`으로 표시한다.
- 이미 회복 확인가와 일봉 돌파 확인가 위에 있으면 회복 확인가는 `이미 회복, 종가 유지 확인으로 대체`로 표시한다.
- 국내 주식 매매 가격은 호가단위에 맞춰 출력한다.
- Markdown 표 안에는 개행 문장을 넣지 않는다.
- 섹터/지표 문장 정밀도와 내부 검증 QA는 docs/02, docs/04, docs/05 규칙을 따른다.

## 5. QA 실패 처리

QA 실패 시 정상 Markdown/HTML 보고서를 만들지 않는다.

QA 실패 시 아래 파일만 저장한다.

```text
reports/종목명_종목코드/종목명_종목코드_보고서_QA실패.md
```

정상 보고서 마지막에는 반드시 `## 내부 검증` 섹션이 있어야 한다.

보고서에는 긴 QA 표를 출력하지 않고 아래 내부 검증 요약 항목만 표시한다.

```text
내부 검증: 통과
가격 신뢰도: 높음/중간/낮음
거래량 신뢰도: 높음/중간/낮음
지표 신뢰도: 높음/중간/낮음
교차검증 완전성: 높음/중간/낮음
수급 신뢰도: 높음/중간/낮음
해석 완전성: 높음/중간/낮음
```

내부 검증 섹션이 없으면 정상 보고서 저장 금지다.

QA 규칙의 상세 내용은 `docs/05_QA_RULES.md`를 따른다.

## 6. 회귀 테스트 실행 규칙

회귀 테스트는 사용자가 명시적으로 요청한 경우에만 실행한다.

예:

```text
회귀 테스트 실행해줘
```

실행 명령:

```bash
python run_regression_tests.py
```

회귀 테스트 대상, PASS/FAIL 기준, 요약 파일 규칙은 `docs/06_REGRESSION_TEST_RULES.md`를 따른다.

## 7. 세부 규칙 문서

세부 규칙 문서:

- 운영 흐름: docs/01_OPERATION_RULES.md
- 매매 판단: docs/02_TRADING_DECISION_RULES.md

- 보고서 오류 수정: 04, 05 확인
- 데이터 수집 오류 수정: 03, 05 확인
- 회귀 테스트 수정: 05, 06 확인

```bash
python analyze_stock_intraday.py 033100 제룡전기
```

완료 일봉 중심 분석:

```bash
python analyze_stock.py 033100 제룡전기
```

```bash
python run_regression_tests.py
```

테스트 대상 확인:

```bash
python run_regression_tests.py --dry-run
```

## 9. 수정 작업 안전 규칙

문서나 코드 수정 요청에서는 아래를 먼저 확인한다.

- 사용자가 분석 실행을 금지했는가?
- 사용자가 회귀 테스트 실행을 금지했는가?
- 새 보고서/차트/CSV 생성을 금지했는가?
- 수정 대상 파일이 제한되어 있는가?

금지된 작업은 절대 실행하지 않는다.

필요 시 정적 문법 확인은 가능하지만, 분석 스크립트 실행은 보고서 생성으로 이어질 수 있으므로 금지 요청이 있으면 피한다.

## 10. 유지해야 하는 파일명

아래 실행 파일명은 바꾸지 않는다.

```text
analyze_stock.py
analyze_stock_intraday.py
run_regression_tests.py
```

기본 테스트 케이스 파일명도 유지한다.

```text
test_cases.yml
```

보고서 기본 폴더도 유지한다.

```text
reports/
```

## 11. 작업 전 체크리스트

작업을 시작하기 전에 아래를 확인한다.

- 요청이 분석 실행인지 문서/코드 수정인지 구분했는가?
- 정규장 중이면 장중 분석 모드가 필요한가?
- 장외이면 완료 일봉 기준 분석이 맞는가?
- 사용자가 재분석 금지를 명시했는가?
- 사용자가 회귀 테스트 금지를 명시했는가?
- 새 보고서, 차트, CSV 생성이 금지되었는가?
- 수정 대상 파일이 제한되어 있는가?
- 세부 규칙 문서를 확인해야 하는가?
- QA 실패 시 정상 보고서 저장 금지 원칙을 지켰는가?
- 현재가가 눌림목 아래일 때 회복 후 지지 확인 문구를 사용했는가?
- 현재가가 깊은 눌림목 구간 안일 때 반등 확인 후 검토 문구를 사용했는가?
- 손익비는 양호하지만 지금 바로 매수 불가인 경우 제한 설명을 붙였는가?
- 장중 주의선 아래에서 보유자 방어 관찰 우선 문구를 사용했는가?
- 종목과 무관한 섹터 문구가 복붙되지 않았는가?
- RSI, MACD, 볼린저밴드 해석이 실제 수치와 일치하는가?
- 주요 지지/저항 가격이 중복 출력되지 않았는가?
- 지금 매수 불가인데 최종 결론 표에 가능으로 표시하지 않았는가?
- 최종 답변 형식이 사용자가 지정한 형식과 일치하는가?

## 12. 작업 후 체크리스트

작업을 마치기 전에 아래를 확인한다.

- 기존 실행 파일명이 유지되었는가?
- 기존 보고서 저장 경로가 유지되었는가?
- 금지된 분석/회귀 테스트/보고서 생성이 실행되지 않았는가?
- 새 문서나 규칙이 AGENTS.MD 참조 구조와 연결되었는가?

## 13. 현대차형 거래량 강세/모멘텀 미회복 규칙

- 현재가가 눌림목 지지가 상단보다 위이고 회복 확인가보다 아래이며, 시간가중 거래량이 1.5배 이상인데 RSI가 50 미만이고 MACD가 신호선 아래이면 `거래량 동반 반등 시도이나 모멘텀 미회복`으로 표시한다.
- 이 경우 거래량 상태는 `좋음`으로 둘 수 있지만 주 전략은 `거래량 강하지만 모멘텀 미회복, 회복 확인 전 추격 금지`로 쓴다.
- 현재가가 20일선 아래, 60일선 위이면 일봉 상태는 `상승` 단독이 아니라 `중기 상승 속 단기 조정`으로 표시한다.
- 회복 확인가와 일봉 돌파 확인가가 같은 가격이면 `회복/돌파 공통 확인가`로 통합하고 행동 문구는 `종가 안착 + 거래량 유지 확인`으로 쓴다.
- 장중 방어선 기준 손익비가 높아도 스윙 손익비 또는 회복/돌파 진입 손익비가 1.2 미만이면 최종 판단에 `스윙/돌파 손익비 부족`을 반영한다.
- 가격/거래량/교차검증 신뢰도가 모두 높으면 `소스 지연 여부 확인`을 오늘 할 행동에 넣지 않는다.

## 14. 한미반도체형 고거래량 음봉/역전 재돌파선 규칙

- 단기 재돌파선은 1차/2차 목표와 역할이 달라야 하며, 단기 재돌파선이 2차 목표보다 높으면 단기 진입선에서 제거하고 `이전 고점/강한 저항`으로 재분류한다.
- MACD 설명은 실제 MACD 수치의 부호를 따른다. MACD < 0이면 `음수권`, MACD > 0이면 `양수권`이며, MACD < 0이고 Signal 위·Histogram 양수이면 `음수권에서 신호선 위로 회복 시도`로 쓴다.
- 거래량 20일 평균 대비 1.5배 이상이면서 음봉이면 `고거래량 음봉/매물 출회 경고`로 표시한다. 거래량 2.0배 이상이고 종가가 고가 대비 크게 밀리면 `강한 매물 출회 경고`를 추가한다.
- 장대 음봉의 증가 거래량은 돌파 매수 근거가 아니다. 이 경우 `거래량은 증가했지만 돌파 매수 근거가 아니라 매물 소화 확인 필요`로 쓴다.
- 스윙 손익비 또는 회복/돌파 진입 손익비가 0.8 미만이면 `신규매수 금지에 가까움`을, 회복/돌파 진입 손익비가 0.5 미만이면 `돌파 매수 전략 성립 불가`를 최종 판단에 반영한다.
- 가격/거래량/교차검증이 정상일 때는 `대표 가격 소스 일치 여부 확인` 같은 데이터 확인 문구를 지금 할 행동에 넣지 않는다.

## 15. 상태코드 기반 판단 구조

보고서는 문장을 먼저 만들고 QA로 뒤늦게 잡는 방식이 아니라 아래 순서를 따른다.

```text
데이터 계산 → 상태 분류 → TradeState 생성 → final_action_state 결정 → 상태코드별 문장 렌더링 → 불변조건 QA → 보고서 저장
```

`TradeState`에는 가격 위치, 추세, 거래량, 캔들, 모멘텀, MACD, RSI, 손익비, 가격 데이터, 거래량 데이터, 수급, 교차검증, 최종 행동 상태코드를 반드시 포함한다.

최종 판단은 `final_action_state` 템플릿으로만 만든다. 보고서 렌더링 중 최종 판단을 임의로 덮어쓰지 않는다.

핵심 상태코드:

- 가격 위치: `BELOW_PULLBACK`, `IN_PULLBACK`, `ABOVE_PULLBACK_BELOW_RECOVERY`, `RECOVERY_TEST`, `ABOVE_RECOVERY_BELOW_BREAKOUT`, `BREAKOUT_INTRADAY_UNCONFIRMED`, `BREAKOUT_CONFIRMED`, `NEAR_TARGET`, `BELOW_DEFENSE`
- 거래량: `LOW_VOLUME`, `NORMAL_VOLUME`, `STRONG_VOLUME_UP`, `STRONG_VOLUME_DOWN`, `HIGH_VOLUME_BEARISH_REVERSAL`, `VOLUME_SPIKE_WARNING`
- MACD: `MACD_POSITIVE_MOMENTUM`, `MACD_NEGATIVE_RECOVERY`, `MACD_WEAKENING`, `MACD_BEARISH`, `MACD_INDETERMINATE`
- RSI: `RSI_OVERHEATED`, `RSI_STRONG`, `RSI_POSITIVE`, `RSI_NEUTRAL_RECOVERY`, `RSI_WEAK_REBOUND`, `RSI_BEARISH`
- 손익비: `RR_GOOD`, `RR_ACCEPTABLE_INTRADAY_ONLY`, `RR_WEAK`, `RR_BAD`, `RR_STRATEGY_INVALID`
- 가격 데이터: `PRICE_DATA_OK`, `PRICE_DATA_STALE_SECONDARY`, `PRICE_DATA_MISMATCH`, `PRICE_DATA_INVALID`
- 거래량 데이터: `VOLUME_DATA_OK`, `VOLUME_DATA_STALE_SECONDARY`, `VOLUME_DATA_MISMATCH`, `VOLUME_DATA_INVALID`
- 수급: `SUPPLY_OK`, `SUPPLY_PARTIAL`, `SUPPLY_MISSING`
- 교차검증: `CROSS_VALIDATION_OK`, `CROSS_VALIDATION_PARTIAL`, `CROSS_VALIDATION_STALE_SECONDARY`, `CROSS_VALIDATION_MISMATCH`, `CROSS_VALIDATION_INVALID`
- 최종 행동: `NO_BUY_DATA_INVALID`, `NO_BUY_BELOW_RECOVERY`, `NO_BUY_BAD_RR`, `NO_BUY_STRATEGY_INVALID`, `NO_BUY_OVERHEATED_BAD_RR`, `NO_BUY_HIGH_VOLUME_BEARISH`, `WAIT_PULLBACK_SUPPORT`, `WAIT_RECOVERY_CLOSE`, `WATCH_INTRADAY_BREAKOUT`, `HOLD_AND_TRAIL`, `PARTIAL_PROFIT_NEAR_TARGET`, `DEFENSE_REQUIRED`

상태코드 불변조건:

- `PRICE_DATA_INVALID`, `VOLUME_DATA_INVALID`, `CROSS_VALIDATION_INVALID`이면 정상 보고서 저장 금지
- `RR_BAD` 또는 `RR_STRATEGY_INVALID`이면 신규매수 긍정 문구 금지
- `RR_STRATEGY_INVALID`이면 반드시 `NO_BUY_STRATEGY_INVALID` 최종 행동 코드를 사용
- 가격/거래량 데이터가 정상이지만 수급이 없으면 `price_data_state = PRICE_DATA_OK`, `volume_data_state = VOLUME_DATA_OK`, `supply_state = SUPPLY_MISSING`로 분리
- 매매 타점 표의 회복/돌파 확인가, 눌림목 지지가, 재돌파선 또는 강한 저항, 1차 목표, 2차 목표, 장중 방어선/장외 안내, 스윙 손절선은 반드시 같은 HTML table 안에 렌더링되어야 함
- `MACD_NEGATIVE_RECOVERY`이면 `음수권에서 신호선 위로 회복 시도` 문구 필수
- `HIGH_VOLUME_BEARISH_REVERSAL`이면 고거래량 음봉/매물 출회 경고 필수
- `final_action_state`와 최종 판단 템플릿이 다르면 QA 실패
- 정상 보고서의 `내부 검증: 통과`는 blocking error가 0개일 때만 허용
