# 07_PROMPTS - 명령 예시

## 국내 종목 분석 명령

```text
제룡전기 033100 분석해
```

일반 종목 분석 요청에서는 해당 종목만 분석한다.

회귀 테스트는 자동 실행하지 않는다.

```bash
python analyze_stock_intraday.py 033100 제룡전기
```

## 대화형 분석 프롬프트

아래처럼 실행하면 프로그램을 켜둔 상태에서 자연어로 분석을 요청할 수 있다.

```bash
python money_assistant.py
```

입력 예시:

```text
삼성전자 분석해줘
005930 삼성전자 장중 분석해줘
삼성전자 일봉 분석해줘
삼성전자 키움 조건부 분석해줘
NVDA 분석해줘
```

기본 모드는 국내 정규장 중이면 장중 분석, 장외이면 일봉/스윙 분석을 실행한다.

`키움`, `조건부`, `명령형`이 포함되면 `command_chart_analyzer.py`를 실행한다.

키움 조건부 명령형 분석은 `KIWOOM_BRIDGE_URL`이 설정되고 로컬 키움 브릿지가 실행 중이어야 한다.

## 완료 일봉 분석 명령

```bash
python analyze_stock.py 033100 제룡전기
```

## 키움 조건부 명령형 차트 분석

```bash
python command_chart_analyzer.py 005930 삼성전자
```

키움 API는 분석 데이터 수집에만 사용하고 주문 기능은 사용하지 않는다.

키움 현재가, 체결, 분봉 데이터 검증에 실패하면 정상 보고서를 저장하지 않고 QA 실패 파일만 저장한다.

## 미국 종목 분석 명령

```text
NVDA 분석해
```

## 평단 포함 분석 요청

```text
제룡전기 033100 평단 55,000원 기준으로 분석해
```

평단가가 제공되면 평단 기준 수익률과 보유자 대응을 추가한다.

```text
회귀 테스트 실행해줘
```

## 코드 수정 요청 예시

```text
이번 작업에서는 종목 재분석을 실행하지 말고 코드와 지침만 수정해라.
수정 대상은 AGENTS.MD, analyze_stock.py, analyze_stock_intraday.py다.
```

이 경우 분석 스크립트를 실행하지 않는다.

가격 관계 판단 수정 요청 예시:

```text
현재가가 돌파 확인가 위인데 돌파 확인가를 기다린다는 문구가 나왔다.
재분석하지 말고 가격 관계 판단 엔진과 QA만 수정해라.
```

호가단위 수정 요청 예시:

```text
보고서에 2,578,450원처럼 호가단위에 맞지 않는 가격이 나왔다.
종목 재분석 없이 호가단위 반올림과 내부 검증만 수정해라.
```

## QA 실패 후 수정 요청 예시

```text
현재 보고서에서 yfinance 1분봉은 실패했는데 yfinance 분봉 정상이라고 표시됐다.
재분석하지 말고 QA와 문구 생성 로직만 수정해라.
```

이 경우 정상 보고서 생성, 차트 생성, CSV 생성을 하지 않는다.

## 문서 구조 수정 요청 예시

```text
AGENTS.MD를 최상위 진입점으로 줄이고 세부 규칙은 docs/로 분리해라.
종목 분석과 회귀 테스트는 실행하지 마라.
```

이 경우 문서만 수정한다.
