# 01_OPERATION_RULES - 운영 흐름

## 전체 실행 흐름

1. 사용자 입력에서 종목명과 코드 또는 티커를 파악한다.
2. 국내/미국 시장을 판별한다.
3. 현재 시간이 해당 시장 정규장인지 확인한다.
4. 정규장 중이면 장중 최신 분석 모드를 우선한다.
5. 정규장 외에는 최신 완료 일봉 종가를 기준으로 한다.
6. 공개 데이터 소스를 2개 이상 수집한다.
7. 가격, 거래량, 날짜, 시장 suffix를 교차검증한다.
8. 차트, 보조지표, 매물대를 계산한다.
9. 매수 관심가, 돌파 확인가, 목표, 방어 기준을 산출한다.
10. 실전 매매 판단 리포트 초안을 생성한다.
11. 단일 보고서 내부 검증을 실행한다.
12. QA 통과 시 정상 보고서를 저장한다.
13. QA 실패 시 정상 보고서는 저장하지 않고 QA 실패 파일만 저장한다.

프로젝트 역할은 아래 구조로 본다.

```text
Stock Analysis Agent
= 데이터 수집기
+ 지표 계산기
+ 가격 관계 판단 엔진
+ 매매 시나리오 생성기
+ 프로 트레이더 판단 엔진
+ 간단 실전 리포트 생성기
+ 단일 보고서 내부 검증기
```

## 장중/장외 판단

국내 정규장:

```text
09:00~15:30 KST
```

미국 정규장:

```text
09:30~16:00 ET
```

장중 분석에서는 장중 현재가, 당일 고가/저가, 장중 누적 거래량을 우선 반영한다.

단, 장중 캔들은 확정 일봉이 아니므로 최종 매수/매도 판단은 오늘 종가로 재확인해야 한다.

장외 분석에서는 최신 완료 일봉 종가를 기준가로 사용한다.

## 국내/미국 시장 분기

국내 종목은 6자리 숫자 코드로 처리한다.

KOSPI/KOSDAQ 시장은 종목 코드와 시장 정보를 바탕으로 판별한다.

미국 종목은 티커 문자열로 처리한다.

국내 종목에는 `.KS` 또는 `.KQ` suffix를 yfinance 보조 확인에만 사용한다.

## 저장 폴더 구조

국내 주식:

```text
reports/종목명_종목코드/
```

미국 주식:

```text
reports/종목명_티커/
```

정상 보고서 파일:

```text
종목명_종목코드_매매타점_분석보고서.md
종목명_종목코드_매매타점_분석보고서.html
종목명_종목코드_데이터검증.csv
종목명_종목코드_지표요약.csv
```

차트 파일은 생성 시 종목 폴더 안에 저장한다.

## QA 실패 처리

QA 실패 시 정상 Markdown/HTML 보고서를 저장하지 않는다.

QA 실패 파일:

```text
종목명_종목코드_보고서_QA실패.md
```

QA 실패 파일에는 실패 사유와 생성 중단된 초안을 포함할 수 있다.

## 실행 금지 상황

사용자가 아래를 명시하면 분석 스크립트를 실행하지 않는다.

- 종목 재분석 금지
- 회귀 테스트 금지
- 새 보고서 생성 금지
- 새 차트 생성 금지
- 새 CSV 생성 금지

일반 종목 분석 요청에서는 해당 종목만 분석하고 회귀 테스트를 자동 실행하지 않는다.

회귀 테스트는 사용자가 명시적으로 요청한 경우에만 선택적으로 실행한다.

단일 보고서 내부 검증은 종목 분석마다 반드시 실행한다.

문서 수정 작업에서는 실행 파일명과 보고서 저장 경로를 변경하지 않는다.
