# 04_REPORT_FORMAT_RULES - 보고서 형식 규칙

## 보고서 섹션 순서

보고서는 장문 설명서가 아니라 `실전 매매 판단 리포트`로 작성한다.

국내/미국, 장중/장외 모두 기본 구조는 아래 6개 섹션이다.

1. 최종 결론
2. 프로 트레이더 관점
3. 차트 분석
4. 보조지표 판단
5. 매매 타점
6. 최종 한 문단

권장 제목:

```text
# 종목명 종목코드 실전 매매 판단 리포트
```

`최종 결론` 표에는 아래 항목만 간단히 표시한다.

```text
현재가
지금 매수
주 전략
회복 확인가 또는 눌림목 지지가
단기 재돌파선
일봉 돌파 확인가
1차 목표
2차 목표
손절/방어
장중 방어선 기준 손익비
스윙 손절선 기준 손익비
회복/돌파 진입 기준 손익비
최종 판단
```

`프로 트레이더 관점`은 6줄 이내로 작성한다.

```text
- 지금 할 행동:
- 지금 하지 말아야 할 행동:
- 신규매수자:
- 보유자:
- 추가매수자:
- 손실보유자:
```

`지금 할 행동`은 추상 문장만 쓰지 말고 가격 알림 중심으로 작성한다.

예:

```text
54,100원 재지지 확인, 57,200원 종가 회복 확인, 60,000원 1차 목표/저항 확인, 49,400원 이탈 시 방어
```

데이터 지연이나 보조 소스 이슈가 있으면 `지금 할 행동`은 아래처럼 나눠 쓴다.

```text
데이터 상태: 보조 소스 지연 여부 확인; 가격 상태: 27,600원 재지지, 27,850원 회복, 28,950원 종가 돌파 확인; 위험 상태: 25,600원 이탈 시 방어
```

`지금 하지 말아야 할 행동`은 상황별로 쓴다.

- RSI 70 이상: `과열권에서 장중 급등 가격을 추격매수하지 않습니다.`
- 손익비 1.0 미만: `손익비가 맞지 않는 돌파 추격매수를 하지 않습니다.`
- 현재가와 1차 목표 차이가 5% 이상이면 `근접 저항 바로 아래`라고 쓰지 않는다.

`차트 분석`은 월봉/주봉/일봉/분봉을 짧게 구분한다.

상태 값은 `상승`, `중립`, `하락`, `참고 제한`, `데이터 부족` 중 하나로 쓴다.

표 안 문장은 반드시 한 줄로 쓴다.

장중 주의 문구처럼 긴 문장은 표 밖에 별도 문장으로 둔다.

`보조지표 판단`은 지표 수치 나열이 아니라 매매 판단과 연결한다.

상태 값은 지표별 숫자 조건으로 산정한다.

RSI 상태 값:

```text
과열 / 강함 / 양호 / 중립 회복 / 약한 반등 / 약세 / 데이터 부족
```

MACD 상태 값:

```text
좋음 / 혼조·개선 중 / 중립 / 중립·개선 시도 / 나쁨 / 데이터 부족
```

`매매 타점`에는 회복 확인가, 눌림목 지지가, 단기 재돌파선, 일봉 돌파 확인가, 1차 목표, 2차 목표, 장중 방어선, 스윙 손절선을 표시한다.

매매 타점 표에는 단기 재돌파선을 별도 행으로 표시한다.
단, 단기 재돌파선이 1차 목표보다 높으면 `단기 재돌파선` 행으로 표시하지 않고 `강한 저항/목표권 확인선` 또는 `강한 저항/2차 목표 전 확인선` 행으로 재분류한다.
이 재분류 가격은 신규매수 기준이 아니라 보유자 익절/비중관리 기준으로만 설명한다.

```text
| 회복 확인가 | 현재가보다 위의 재진입 확인 가격 |
| 눌림목 지지가 | 현재가 부근 또는 아래의 지지 확인 가격 |
| 단기 재돌파선 | 위 유지 시 단기 탄력 확인 |
| 일봉 돌파 확인가 | 종가 유지/안착과 거래량 확인 |
```

현재가보다 위에 있는 재진입 가격을 `눌림목` 또는 `매수 관심가` 단일 명칭으로 표시하지 않는다.

현재가가 눌림목 지지가보다 위이고 일봉 돌파 확인가보다 아래이면 `회복 확인가`를 `해당 없음`으로 쓰지 않는다.

단기 재돌파선과 1차 목표가 같으면 단기 재돌파선 가격은 별도 출력하지 않고 `1차 목표/강한 저항과 중복되어 별도 표시 생략`으로 쓴다.

현재가가 이미 돌파 확인가 위이면 행동 문구는 `종가 유지와 거래량 증가 확인`으로 쓴다.

현재가가 단기 재돌파선과 같으면 행동 문구는 `위 유지 시 단기 탄력 확인`으로 쓴다.

장중 현재가가 단기 재돌파선과 같으면 가격 칸에 `장중 현재가 유지 기준`을 포함하고, 행동 문구는 아래처럼 쓴다.

```text
현재가 부근 위 유지 시 단기 탄력 유지, 이탈 시 장중 돌파 실패 가능성 확인
```

현재가가 이미 회복 확인가와 일봉 돌파 확인가 위라면 회복 확인가 가격 칸은 `이미 회복, 종가 유지 확인으로 대체`로 쓰고, 행동 칸은 `신규 진입 기준은 오늘 종가 유지 확인으로 대체`로 쓴다.

장중 현재가가 일봉 돌파 확인가 위여도 `돌파 확정`, `조건 충족 매수 가능`으로 쓰지 않고 `장중 돌파 시도`, `종가 유지 확인 필요`로 표현한다.

`최종 한 문단`은 3~5문장 이내로 쓴다.

보고서 길이는 Markdown 기준 120줄 이내를 권장한다.

긴 데이터 출처 목록, 긴 QA 표, 트레이딩 점수 상세표, A/B/C/D 시나리오 표는 기본 보고서에 출력하지 않는다.

차트와 CSV 산출물 생성 흐름은 유지하되, 보고서 본문은 행동 중심으로 축약한다.

최종 보고서 구조는 아래만 사용한다.

```text
# 종목명 종목코드 실전 매매 판단 리포트

## 1. 최종 결론
## 2. 프로 트레이더 관점
## 3. 차트 분석
## 4. 보조지표 판단
## 5. 매매 타점
## 6. 최종 한 문단
## 내부 검증
```

내부 검증에는 아래 신뢰도 항목을 분리해서 표시한다.

```text
가격 신뢰도:
거래량 신뢰도:
지표 신뢰도:
교차검증 완전성:
수급 신뢰도:
해석 완전성:
```

pykrx/FDR이 일치하고 yfinance만 최신거래일이 지연되면 `교차검증 완전성: 중간`으로 표시하고, 가격/거래량 신뢰도를 무조건 낮음으로 낮추지 않는다.

정밀 판단 중단 상태에서 매매 타점 표를 출력해야 한다면 회복 확인가, 눌림목 지지가, 일봉 돌파 확인가, 1차/2차 목표, 스윙 손절선 앞에 `참고` 라벨을 붙인다.

콘솔 출력은 아래 형식으로 제한한다.

```text
[분석 완료]

종목:
현재가:
지금 매수:
주 전략:
회복 확인가 또는 눌림목 지지가:
일봉 돌파 확인가:
1차 목표:
손절/방어:
최종 판단:
데이터 신뢰도:
보고서 경로:
```

## 알림 설정 가격 섹션

섹션명은 반드시 아래처럼 쓴다.

```text
## 알림 설정 가격
```

아래 섹션명은 쓰지 않는다.

```text
## 눌림 설정 가격
```

표 헤더는 아래처럼 쓴다.

```text
| 알림 가격 | 의미 | 행동 |
```

아래 헤더는 쓰지 않는다.

```text
| 눌림 가격 | 의미 | 행동 |
```

알림 설정 가격에는 눌림목, 단기 재돌파선, 일봉 돌파 확인선, 장중 방어 가격, 스윙 최종 방어 가격을 함께 표시할 수 있다.

## 데이터 신뢰도 문구

데이터 신뢰도 표에서는 같은 상태 접두어를 반복하지 않는다.

금지:

```text
장중 장중 경고
장중 장중 통과
분봉 분봉 경고
완료 일봉 완료 일봉 통과
```

사용:

```text
장중 경고
장중 통과
분봉 경고
완료 일봉 통과
```

## 손익비 제한 설명

최종 판단 문장에서는 손익비와 매수 가능 여부가 충돌하지 않도록 제한 설명을 붙인다.

지금 바로 매수 불가인데 손익비가 양호하면 아래 의미를 포함한다.

```text
손익비는 양호하지만 지금 바로 신규매수 조건은 충족하지 못했습니다.
```

## 섹터/지표 문장 정밀도

실패 조건 문구에는 종목과 무관한 섹터명을 쓰지 않는다.

예를 들어 HPSP에는 전력기기/변압기 문구를 쓰지 않고 `반도체 장비/공정 장비` 또는 `동종 업종`을 사용한다.

RSI, MACD, 볼린저밴드 해석 문구는 반드시 실제 수치 조건과 일치해야 한다.

- RSI가 50 이상이면 `50 아래`라고 쓰지 않는다.
- MACD가 신호선 위이고 히스토그램 양수이면 `개선 확인 전`이라고 쓰지 않는다.
- 현재가가 볼린저밴드 중심선보다 충분히 위이면 `중심선 근처`라고 쓰지 않는다.

주요 지지/저항 가격은 중복 표시하지 않는다.

금지:

```text
주요 저항: 92,000원, 92,000원
주요 지지: 51,300원, 51,300원
```

## 한글 라벨 규칙

영어 제목 대신 한글 표현을 사용한다.

예:

| 금지 | 사용 |
|---|---|
| Final Rating | 최종 판단 |
| Risk/Reward | 손익비 |
| Stop-loss | 손절 |
| Defense line | 방어선 |
| Pullback entry | 눌림목 매수 |
| Breakout entry | 돌파 매수 |

## 목표/익절 라벨 분리

목표/익절 표현은 아래 3개로 통일한다.

```text
근접 저항/보유자 일부 익절 후보
신규매수 기준 1차 목표
신규매수 기준 2차 목표
```

아래 표현은 보고서에 쓰지 않는다.

```text
1차 익절 가격
2차 익절 가격
1차 목표가
2차 목표가
```

## 방어선 라벨 분리

장중 보고서에서는 아래 4개 라벨을 분리한다.

```text
장중 주의선
장중 방어선
스윙 최종 방어선
전량 이탈 조건
```

장중 보고서에서 `주의선`, `방어선` 단독 라벨을 쓰지 않는다.

## 장초 시간가중 거래량 문구

09:30 이전에는 시간가중 환산 거래량을 확정 신호로 해석하지 않는다.

필수 문구:

```text
장초반 30분 이내 시간가중 환산 거래량은 과장될 수 있으므로 참고값으로만 봅니다.
```

금지 표현:

```text
거래량 급증 확정
강한 거래량 확인
거래량 동반 확정
```

## 분봉 상태 문구

yfinance 1분봉이 실패하면 `정상`이라고 쓰지 않는다.

권장:

```text
yfinance 분봉: 수집 실패 또는 데이터 없음
분봉 데이터 신뢰도: 부분 통과
```

## 금지 표현

보고서에서는 사용자가 헷갈릴 수 있는 혼합 표현을 피한다.

- 눌림목 매수만 가능이라고 하면서 동시에 돌파 매수 가능처럼 표현
- 현재가가 눌림목 아래인데 단순히 눌림목 지지 중이라고 표현
- 당일 고가 재돌파와 일봉 돌파 확인선을 같은 급으로 묶는 표현
- 같은 가격을 `343,000원~343,000원`처럼 범위로 표시
- 수급 실패 상태에서 수급 방향 단정
- 분봉 실패 상태에서 분봉 정상 표시

## 회복/돌파 공통 확인가 형식

회복 확인가와 일봉 돌파 확인가가 같은 가격이면 두 줄로 반복하지 않고 아래처럼 한 줄로 쓴다.

```text
회복/돌파 공통 확인가: 624,000원
행동: 종가 안착 + 거래량 유지 확인
```

역할 분리:

- 눌림목 지지가: 신규매수 대기 구간
- 회복/돌파 공통 확인가: 종가 회복과 돌파를 동시에 확인하는 가격
- 단기 재돌파선: 장중 고점 재돌파 확인
- 1차 목표: 보유자 일부 익절 후보
- 장중 방어선: 장중 실패 판단
- 스윙 손절선: 일봉 종가 기준 방어

가격/거래량/교차검증 신뢰도가 모두 높으면 `오늘 할 행동`에 `소스 지연 여부 확인`을 넣지 않는다. 이 문구는 yfinance 지연, 최신거래일 불일치, 가격 불일치, 거래량 불일치가 있을 때만 쓴다.

## 고거래량 음봉/역전 재돌파선 출력

거래량 표시는 거래량 배율만 쓰지 않고 캔들 방향을 함께 쓴다.

- 양봉 + 거래량 1.5배 이상: `거래량 동반 상승`
- 음봉 + 거래량 1.5배 이상: `고거래량 음봉/매물 출회 경고`
- 음봉 + 거래량 2.0배 이상 + 고가 대비 큰 밀림: `강한 매물 출회 경고`

단기 재돌파선이 2차 목표보다 높으면 `단기 재돌파선` 가격 칸에 해당 가격을 그대로 쓰지 않는다. 가격은 별도 `이전 고점/강한 저항` 행으로 이동하고, 단기 재돌파선은 `이전 고점/강한 저항으로 재분류`라고 표시한다.

MACD 설명에는 실제 수치 부호와 맞는 `양수권` 또는 `음수권`만 사용한다.

손익비 경고는 최종 문단에서 한 문장으로 압축한다.

예:

```text
스윙 손익비 0.45배, 회복/돌파 진입 손익비 0.18배로 손익비 부족이며 신규매수와 돌파 추격매수 모두 부적합하고 돌파 매수 전략 성립 불가입니다.
```

## 상태코드 출력 섹션

정상 보고서에는 최종 결론 다음에 `상태코드 기반 판단` 섹션을 넣는다.

형식:

```text
## 1-1. 상태코드 기반 판단

| 상태 항목 | 상태코드 |
|---|---|
| 가격 위치 | ... |
| 추세 | ... |
| 거래량 | ... |
| 캔들 | ... |
| 모멘텀 | ... |
| MACD | ... |
| RSI | ... |
| 손익비 | ... |
| 가격 데이터 | ... |
| 거래량 데이터 | ... |
| 수급 | ... |
| 교차검증 | ... |
| 최종 행동 | ... |
```

매매 타점 표의 `회복/돌파 확인가`, `눌림목 지지가`, `재돌파선 또는 강한 저항`, `1차 목표`, `2차 목표`, `장중 방어선/장외 안내`, `스윙 손절선`은 같은 Markdown/HTML table 안에 포함되어야 한다.
빈 줄 때문에 필수 행이 별도 표나 HTML 문단으로 분리되면 정상 보고서를 저장하지 않는다.

사용자 유형별 대응 문구는 상태코드 렌더링 결과를 우선 사용한다.

- 신규매수자: `new_buyer_action`
- 보유자: `holder_action`
- 추가매수자: `add_buyer_action`
- 손실보유자/방어: `stop_loss_action`

보고서 렌더링 단계에서는 `final_action_state`가 만든 최종 판단을 덮어쓰지 않는다.
