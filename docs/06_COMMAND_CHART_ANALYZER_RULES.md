# 조건부 명령형 차트 분석 규칙

## 목적

이 문서는 기존 `analyze_stock.py`의 설명형 리포트와 별도로, 사용자가 실제 매매 판단에 바로 쓸 수 있는 **조건부 명령형 차트 분석기**의 판정 규칙을 정의한다.

자동매매, 자동 주문, 주문 API 호출은 포함하지 않는다.

## 최종 출력 원칙

분석기는 아래처럼 애매한 문장을 쓰지 않는다.

> 추세는 양호하나 눌림 확인 후 접근이 유리합니다.

대신 아래처럼 가격과 행동을 같이 출력한다.

> 지금은 사지 마라. 49,000원을 깨지 않고 49,300원 이상에서 3분봉 또는 5분봉 종가가 마감될 때만 1차 매수하라.

## 가격 산출 근거

분석기가 제시하는 가격은 반드시 아래 지표 중 하나 이상에서 산출되어야 한다.

| 가격 종류 | 근거 |
|---|---|
| 핵심 지지선 | 20일선, 60일선, 볼린저밴드 중심선/하단, 일목 기준선, 일목 구름 상단/하단, 최근 저점 |
| 매수 확인선 | 3분봉/5분봉 20이평선, 3분봉/5분봉 볼린저 중심선, 직전 반등 고점, 5일선, 10일선 |
| 돌파 매수선 | 기존 돌파 확인가, 최근 5일 고점, 최근 20일 고점, 볼린저밴드 상단, 일목 구름 상단 |
| 추격 금지선 | 볼린저밴드 상단 99% 부근, RSI 과열권, 목표가 근접 구간 |
| 손절/방어선 | 기존 `nearest_levels()`의 defense 값, 일목 구름 하단, 주요 이평선 이탈 기준 |

## 판정값

최종 판정은 다음 중 하나로 제한한다.

| 판정 | 의미 |
|---|---|
| 사라 | 현재가 또는 조건 충족 가격에서 1차 분할 매수 가능 |
| 사지 마라 | 지지 실패, 데이터 불일치, 손익비 부족, 회복 전 상태 |
| 기다려라 | 지지선은 살아 있으나 매수 확인선 회복 전 |
| 관망하라 | 지지/돌파 둘 다 아닌 중간 가격 |
| 팔아라 | 방어선 이탈 또는 매수 관점 폐기 |

## 매수 조건 원칙

### 지지선 터치만으로 매수 금지

핵심 지지선에 닿았다는 이유만으로 매수하지 않는다.

필수 조건:

1. 핵심 지지선을 3분봉 또는 5분봉 종가 기준으로 이탈하지 않아야 한다.
2. 매수 확인선 위에서 3분봉 또는 5분봉 종가가 마감되어야 한다.
3. 일목 구름 아래 또는 20일선 아래에서는 회복 확인 전 매수하지 않는다.
4. 볼린저밴드 상단 근처에서는 신규 추격 매수하지 않는다.

### 예시

```text
49,000원은 20일선, 볼린저밴드 중심선, 일목 기준선이 겹치는 핵심 지지선입니다.
따라서 49,000원 부근까지 하락하더라도 바로 매수하지 않습니다.

49,000원을 깨지 않고 49,300원 이상에서 3분봉 또는 5분봉 종가가 마감될 때만 1차 매수합니다.
49,000원 아래에서 3분봉 또는 5분봉 종가가 마감되면 지지 실패로 보고 사지 않습니다.
```

## 데이터 신뢰도 원칙

데이터 신뢰도가 낮거나 대표 가격 산정이 불가능하면 가격 조건을 확정하지 않는다.

출력:

```text
최종 판정: 사지 마라
사유: 데이터 검증 실패. 가격 조건을 확정하지 마라.
```

## 현재 키움 Provider 구조

조건부 명령형 차트 분석기는 이미 로컬 키움 브릿지를 데이터 Provider로 사용한다.

사용 경로:

1. 장중 현재가: `KiwoomDataProvider.get_quote()`
2. 3분봉/5분봉: `KiwoomDataProvider.get_intraday_ohlcv()`
3. 체결 기반 분봉 fallback: `ticks_to_ohlcv()`
4. 일봉 검증: 현재는 pykrx, FinanceDataReader, yfinance 공개 데이터 교차검증을 유지한다.

현재 구조에서 키움 API는 분석 데이터 수집 전용이다.

주문 전송, 자동매매, 시장가 주문, 조건주문 기능은 구현하지 않는다.

키움 브릿지 현재가, 체결, 분봉 검증이 실패하면 공개 데이터로 조용히 대체해 정상 리포트를 저장하지 않고 QA 실패 파일만 저장한다.

## 향후 개선

`KiwoomDataProvider.get_daily_ohlcv()`는 일봉 대표 후보에 추가하되, 키움 일봉 단독 사용은 금지한다.

키움 일봉과 pykrx/FDR 최신 완료 일봉을 교차검증하여 가격 차이가 1% 초과이면 경고, 3% 초과이면 분석 중단 처리한다.
