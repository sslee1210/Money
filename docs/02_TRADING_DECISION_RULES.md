# 02_TRADING_DECISION_RULES - 매매 판단 규칙

## 프로 트레이더 판단 원칙

보고서는 지표 설명서가 아니라 조건부 매매 계획이어야 한다.

항상 아래 질문에 답한다.

- 지금 해야 할 행동은 무엇인가?
- 지금 하지 말아야 할 행동은 무엇인가?
- 어느 가격에서 진입할 것인가?
- 어느 가격이 깨지면 분석이 틀렸다고 볼 것인가?
- 진입한다면 어느 비중으로 들어갈 것인가?
- 보유자는 어디서 익절하고 어디서 방어할 것인가?

종목별 섹터 문구는 코드와 종목명을 기준으로 자동 판별한다.

전력기기, 반도체 장비, 조선, 바이오 등 특정 섹터명은 다른 종목 보고서에 복사해서 쓰지 않는다.

판별이 어렵다면 `동종 업종`이라고 중립적으로 표현한다.

## 가격 관계 판단 엔진

현재가, 회복 확인가/눌림목 지지가, 단기 재돌파선, 일봉 돌파 확인가, 1차 목표, 방어선의 상대 위치를 먼저 판정한 뒤 문장을 만든다.

필수 판정값:

- 현재가가 회복 확인가 아래인지, 눌림목 지지가 안/위인지
- 현재가가 단기 재돌파선 아래/같음/위인지
- 현재가가 돌파 확인가 아래/같음/위인지
- 현재가가 1차 목표에 3% 이내로 가까운지
- 현재가가 방어선에 3% 이내로 가까운지
- 매수 관심가 상단과 돌파 확인가가 사실상 같은 구간인지

가격 관계별 문장:

| 상황 | 문장 |
|---|---|
| 현재가가 돌파 확인가 아래 | 현재가는 돌파 확인가 아래에 있어 돌파 매수 조건이 아직 충족되지 않았습니다. |
| 현재가가 돌파 확인가 위 | 현재가는 돌파 확인가 위에 있으나, 종가 유지와 거래량 동반 여부가 확인되어야 합니다. |
| 현재가와 단기 재돌파선이 같음 | 현재가 부근 위 유지 시 단기 탄력 유지, 이탈 시 장중 돌파 실패 가능성 확인이 필요합니다. |
| 현재가가 눌림목 지지가 위 | 현재가는 지지가보다 위에 있어 추격매수보다 눌림 재형성을 기다립니다. |
| 현재가가 눌림목 지지가 안 | 현재가는 눌림목 지지가 구간 안에 있으므로 지지 확인 후 분할매수를 검토합니다. |
| 현재가가 회복 확인가 아래 | 현재가는 회복 확인가 아래에 있어 반등 회복 확인 전 신규매수는 제한합니다. |
| 현재가가 1차 목표 근처 | 현재가는 1차 목표에 가까워 신규매수보다 보유자 일부 익절 관찰이 우선입니다. |

금지 표현:

- 현재가가 돌파 확인가 위인데 `돌파 확인가를 기다립니다`
- 현재가가 돌파 확인가 위인데 `종가 안착 확인이 먼저입니다`
- 현재가와 단기 재돌파선이 같은데 `단기 재돌파선을 회복해야 합니다`

## 지금 매수 가능 기준

지금 매수 가능은 매우 엄격하게 판단한다.

대부분의 조건이 필요하다.

- 데이터 신뢰도 중간 이상
- 현재가가 주요 이동평균선 위
- 일목 구름 위 또는 구름 돌파 직후
- MACD 개선
- RSI 50 이상
- 거래량이 20일 평균 이상 또는 돌파 시 1.2배 이상
- 손익비 1.5 이상
- 위쪽 매물대 부담이 과도하지 않음

조건이 부족하면 지금 바로 매수는 불가다.

## 눌림목 매수 기준

눌림목 매수는 지지 확인이 핵심이다.

- 주요 지지선 접근
- 거래량 감소 조정
- 아래꼬리 또는 지지선 회복
- 일봉 종가가 지지선 위 마감
- RSI 40~55 구간 회복
- 손절선이 가까워 손익비 개선

현재가가 눌림목 하단보다 아래에 있으면 단순히 “지지 확인”이라고 쓰지 않는다.

이때는 아래처럼 표현한다.

- 눌림목 구간 회복 후 지지 확인
- 눌림목 구간 재진입 후 지지 확인
- 눌림목 재진입 실패 시 신규매수 금지

`눌림목 지지 중`처럼 이미 지지받고 있는 듯한 표현은 금지한다.

현재가가 깊은 눌림목 구간 안에 있으면 보고서에 반드시 `깊은 눌림목 구간 안`임을 명시한다.

깊은 눌림목 안에 있어도 장중 방어선과 가까우면 신규매수는 반등 확인 후에만 검토한다.

권장 표현:

```text
현재가는 깊은 눌림목 구간 안에 있으나, 장중 방어선과 가까워 신규매수는 반등 확인 후에만 검토합니다.
```

현재가가 얕은 눌림목과 깊은 눌림목에 동시에 들어가면 `얕은 눌림목과 깊은 눌림목이 겹치는 구간`이라고 표현한다.

겹침 구간에서는 과도하게 깊은 하락처럼 단정하지 말고, 전일 종가/장중 주의선/장중 방어선과의 위치를 함께 설명한다.

## 돌파 매수 기준

돌파 매수는 종가와 거래량이 핵심이다.

- 주요 저항선 위 일봉 종가 마감
- 거래량 20일 평균 1.2배 이상
- 강한 돌파는 1.5배 이상
- MACD 히스토그램 증가
- RSI 55 이상
- 돌파 다음 날 저항선이 지지선으로 바뀌는지 확인

장중 단기 재돌파는 분봉상 회복 신호일 뿐 신규 돌파 매수 조건과 분리한다.

이미 현재가가 돌파 확인가 위에 있으면 신규 돌파 매수 문구는 `대기`가 아니라 `종가 유지와 거래량 확인`으로 쓴다.

매수 관심가와 돌파 확인가는 역할이 달라야 한다.

| 가격 | 역할 |
|---|---|
| 회복 확인가 | 현재가보다 위에 있는 재진입 확인 가격 |
| 눌림목 지지가 | 현재가 부근 또는 아래의 지지 확인 가격 |
| 단기 재돌파선 | 장중 고점 또는 단기 탄력 회복 가격 |
| 일봉 돌파 확인가 | 종가 유지 확인이 필요한 저항 돌파 구간 |

매수 관심가 상단과 돌파 확인가가 같은 가격이면 내부 검증 실패다.

## 투자자 상태별 대응

보고서는 아래 4개 관점을 분리한다.

| 투자자 상태 | 대응 |
|---|---|
| 신규매수자 | 지금 진입 가능 여부와 기다릴 가격 |
| 기존 보유자 | 보유, 일부 익절, 방어 기준 |
| 추가매수자 | 불타기/물타기 가능 여부 |
| 손실 보유자 | 반등 매도, 손절, 방어 기준 |

평단가가 없으면 개인별 수익률 판단은 제외한다고 표시한다.

## 매매 시나리오

매매 시나리오는 내부 판단에는 사용할 수 있지만 기본 보고서에 A/B/C/D 표로 강제 출력하지 않는다.

기본 보고서에서는 아래 행동만 짧게 제시한다.

- 회복 확인가
- 눌림목 지지가
- 단기 재돌파선
- 일봉 돌파 확인가
- 1차 목표
- 2차 목표
- 장중 방어선
- 스윙 손절선
- 이 가격에서 해야 할 행동

## 트레이딩 점수 시스템

트레이딩 점수는 내부 보조 판단으로 계산할 수 있다.

기본 보고서에는 상세 점수표를 출력하지 않는다.

| 항목 | 점수 |
|---|---:|
| 추세 점수 | 0~20 |
| 모멘텀 점수 | 0~15 |
| 거래량 점수 | 0~15 |
| 손익비 점수 | 0~20 |
| 시장/섹터 점수 | 0~10 |
| 수급 점수 | 0~10 |
| 위치 점수 | 0~10 |

손익비 점수는 아래 3개를 분리해서 계산할 수 있다.

- 현재가 기준 손익비 점수: 현재 위치에서 바로 진입할 경우의 손익비
- 눌림목 진입 기준 손익비 점수: 눌림목 가격에서 진입할 경우의 손익비
- 돌파 진입 기준 손익비 점수: 돌파 확인 후 진입할 경우의 손익비

최종 손익비 점수는 세 점수의 평균 또는 상황별 가중치로 계산한다.

지금 바로 매수가 불가이면 현재가 기준 손익비 점수는 최대 14점으로 제한한다.

지금 바로 매수 불가인데 현재가 기준 손익비가 1.5 이상이면, 손익비는 양호하지만 매수 조건은 미충족이라고 설명한다.

손익비가 좋다는 이유만으로 바로 매수 가능으로 해석하지 않는다.

낮은 손익비 해석:

- 장중 방어선 기준 손익비가 1.2 미만이면 `장중 신규 진입 매력 낮음`으로 표시한다.
- 스윙 손절선 기준 손익비가 1.2 미만이면 `손익비 부족`으로 표시하고 신규매수 매력 낮음을 명시한다.
- 스윙 손절선 기준 손익비가 1.0 미만이면 `스윙 신규매수 부적합`으로 표시한다.
- 회복 확인가 또는 돌파 확인가 진입 기준 손익비가 1.0 미만이면 `돌파 추격매수 부적합`으로 표시한다.
- RSI가 70 이상이고 손익비가 1.2 미만이면 최종 판단에 `과열 추격 금지`를 포함한다.
- 손익비가 1.0 전후이거나 낮은데 최종 판단을 긍정적으로 쓰지 않는다.

## 장중 돌파/과열 판단

장중 현재가가 일봉 돌파 확인가 위에 있어도 오늘 종가가 확정되기 전에는 돌파 확정으로 보지 않는다.

장중 표현:

- 장중 돌파 시도
- 종가 유지 확인 필요
- 종가 유지 전 신규매수 보류

금지 표현:

- 돌파 확정
- 조건 충족 매수 가능
- 장중 돌파만으로 조건 충족 시 분할 매수

과열 돌파 신규매수 부적합 조건:

- current_price > daily_breakout_line
- weighted_volume_ratio >= 1.2
- RSI >= 70
- 회복/돌파 진입 기준 손익비 < 1.0 또는 스윙 손절선 기준 손익비 < 1.0

이 조건에서는 `돌파 유지 확인`보다 아래 판단을 우선한다.

```text
과열 추격 금지·손익비 부족으로 신규 추격매수 부적합
```

최종 문단에는 아래 의미를 포함한다.

```text
장중 돌파와 거래량은 긍정적이나 RSI 과열과 손익비 부족으로 신규 추격매수는 부적합합니다. 보유자는 종가 유지 여부를 확인하고, 신규자는 눌림 또는 종가 확정 후 재판단합니다.
```

`돌파 유지 확인`은 보유자 관점에서만 사용하고, 신규매수자 관점에서는 `종가 유지 전 신규매수 보류` 또는 `신규 추격매수 부적합`으로 쓴다.

거래량 점수는 단순 시간가중 거래량만으로 높게 주지 않는다.

아래 상태에서는 거래량 점수를 감점한다.

- 장중 고점 돌파 후 현재가가 돌파선 아래로 밀림
- 거래량 증가가 상승 지속이 아니라 윗꼬리/돌파 실패로 연결됨
- 현재가가 일봉 돌파 확인선 아래
- 현재가가 단기 재돌파 확인선 아래

시간가중 거래량이 높아도 돌파 유지 실패 상태이면 거래량 점수는 최대 12점으로 제한한다.

돌파 실패, 거래량 증가, 현재가가 돌파선 아래인 조건이 겹치면 거래량 점수는 8~11점 범위로 조정한다.

거래량 점수 14~15점은 돌파 유지 또는 강한 종가 안착이 확인될 때만 허용한다.

## 최종 판단 우선순위

최종 판단은 아래 우선순위로 결정한다.

1. 데이터 신뢰도 낮음: 매수 판단 제한
2. 현재가가 장중 방어선 근처: 방어 우선
3. 현재가가 1차 목표 근처: 신규매수보다 보유자 익절 관찰
4. 현재가가 장중 방어선 아래: 신규매수 금지, 장중 방어 우선
5. 현재가가 매수 관심가 하단보다 아래: 회복 확인 대기
6. 현재가가 매수 관심가 안: 눌림목 지지 확인 후 분할매수 검토
7. 현재가가 매수 관심가 위이고 돌파 확인가 아래: 추격 금지, 눌림 재형성 대기
8. 현재가가 돌파 확인가 위: 종가 유지와 거래량 확인
9. 돌파 확인가 종가 안착 + 거래량 증가: 돌파매수 검토

가격 위치별 전략 라벨:

| 조건 | 주 전략 | 최종 판단 |
|---|---|---|
| 현재가 < 장중 방어선 | 신규매수 금지, 장중 방어 우선 | 방어 확인 전 매수 금지 |
| 장중 방어선 <= 현재가 < 매수 관심가 하단 | 회복 확인 대기 | 매수 관심가 회복 전 신규매수 금지 |
| 매수 관심가 하단 <= 현재가 <= 매수 관심가 상단 | 눌림목 지지 확인 | 지지 확인 시 분할매수 검토 |
| 매수 관심가 상단 < 현재가 < 돌파 확인가 | 추격 금지, 눌림 재형성 대기 | 돌파 전 추격매수 제한 |
| 현재가 >= 돌파 확인가 | 종가 유지와 거래량 확인 | 돌파 유지 확인 |

현재가보다 위에 있는 매수 관심가는 `눌림목`이 아니라 `회복 확인가`로 표시한다.

회복 확인가 산정:

- 현재가가 눌림목 지지가보다 위이고 일봉 돌파 확인가보다 아래이면 회복 확인가가 필요하다.
- 회복 확인가는 20일선, 볼린저밴드 중심선, 일봉 돌파 확인가 중 현재가 위에 있는 가장 가까운 주요 회복선으로 산정한다.
- 가까운 회복선이 0.3% 이내로 붙어 있으면 범위로 표시할 수 있다.
- 이 구조에서 회복 확인가를 `해당 없음`으로 표시하지 않는다.

단기 재돌파선과 목표가 중복 처리:

- 단기 재돌파선은 진입/회복 확인선이고 1차 목표는 익절 후보이므로 같은 가격으로 중복 출력하지 않는다.
- 단기 재돌파선과 1차 목표가 같으면 단기 재돌파선은 `1차 목표/강한 저항과 중복되어 별도 표시 생략`으로 처리한다.
- 단기 재돌파선이 1차 목표보다 높으면 신규매수 기준이 아니라 `강한 저항/목표권 확인선`으로 재분류한다.
- 단기 재돌파선이 2차 목표에 근접하면 보유자 익절/비중관리 기준으로만 출력하고, `단기 재돌파선` 라벨을 쓰지 않는다.
- 해당 가격은 1차 목표 또는 강한 저항으로만 관리한다.

단기 재돌파선이 현재가와 같은 경우:

- 가격 칸에는 `장중 현재가 유지 기준`을 포함한다.
- 행동 문구는 `현재가 부근 위 유지 시 단기 탄력 유지, 이탈 시 장중 돌파 실패 가능성 확인`으로 쓴다.
- `현재가는 단기 재돌파선에 걸쳐 있으므로` 같은 기계적 문장은 쓰지 않는다.

이미 회복 완료 상태:

- 현재가가 회복 확인가와 일봉 돌파 확인가 위에 있으면 회복 확인가를 `해당 없음`으로 쓰지 않는다.
- 가격 칸에는 `이미 회복, 종가 유지 확인으로 대체`를 쓴다.
- 행동 칸에는 `신규 진입 기준은 오늘 종가 유지 확인으로 대체`를 쓴다.

보유자 대응:

- 20일선, 볼린저 중심선, 일봉 돌파 확인가 역할의 회복 확인가는 일부 익절가가 아니다.
- 보유자 문장은 `회복 확인가 회복 실패 시 추가매수 보류`, `눌림목 지지가 재이탈 시 단기 비중 축소 검토`, `1차 목표 접근 시 1차 익절 검토`, `스윙 손절선 이탈 시 방어/손절`처럼 분리한다.
- 회복 확인가 회복 실패만으로 바로 일부 비중 축소라고 쓰지 않는다.

## 지표 해석 조건문

RSI는 아래처럼 구간별로 해석한다.

| RSI | 해석 |
|---:|---|
| 70 이상 | 과열권 진입. 추격매수보다 분할익절 또는 눌림 확인 우선 |
| 60 이상 70 미만 | 강한 모멘텀 구간. 과열 접근 여부 확인 |
| 55 이상 60 미만 | 양호. 추가 탄력 확인 필요 |
| 50 이상 55 미만 | 중립 회복. 강한 매수 신호로 과대평가하지 않음 |
| 40 이상 50 미만 | 약한 반등 시도 구간. 50선 회복 확인 필요 |
| 40 미만 | 약세 구간. 반등 확인 전 신규매수 제한 |

MACD는 MACD, Signal, Histogram의 실제 위치로 해석한다.

| 상태 | 해석 |
|---|---|
| MACD > Signal, Histogram > 0, 현재가 >= 돌파 확인가 | 좋음 |
| MACD > Signal, Histogram > 0, 현재가 < 돌파 확인가 | 혼조/개선 중 |
| MACD > Signal, Histogram <= 0 | 중립 |
| MACD <= Signal, Histogram > 0 | 중립/개선 시도 |
| MACD <= Signal, Histogram <= 0 | 나쁨 |

RSI 50~55 구간은 `좋음`으로 표시하지 않는다.

볼린저밴드는 현재가와 중심/상단/하단 위치를 비교해 해석한다.

| 위치 | 해석 |
|---|---|
| 현재가 > 상단 | 상단 돌파 또는 과열 구간. 윗꼬리와 거래량 확인 |
| 중심선 < 현재가 <= 상단 | 중심선 위 반등 흐름 유지. 상단 접근 전 저항 돌파 확인 |
| 중심선 ±2% | 중심선 근처 공방 구간 |
| 하단 <= 현재가 < 중심선 | 중심선 아래. 반등 확인 필요 |
| 현재가 < 하단 | 하단 이탈 또는 과매도 구간. 반등 확인 전 신규매수 제한 |

주요 지지/저항 가격은 출력 전 중복 제거한다.

동일 가격이 2번 이상 나오면 한 번만 표시한다.

판정:

| 점수 | 판단 |
|---:|---|
| 85~100 | 매우 우수. 단, 조건 충족 시에만 진입 |
| 70~84 | 양호. 분할 진입 가능 |
| 55~69 | 관망 우위. 조건 확인 필요 |
| 40~54 | 매수 금지. 반등 확인 후 대기 |
| 0~39 | 위험. 방어 또는 관망 |

점수가 높아도 현재 위치가 나쁘면 지금 바로 매수 가능과 분리한다.

현재가가 장중 주의선 아래이면 보유자는 신규매수보다 방어 관찰이 우선이다.

권장 표현:

```text
보유자는 신규매수보다 방어 관찰이 우선입니다.
```

## 오늘 할 행동

보고서 상단에는 오늘 해야 할 행동을 번호로 제시한다.

예:

```text
1. 지금 바로 매수하지 않는다.
2. 눌림 지지 여부를 확인한다.
3. 종가 돌파 여부를 확인한다.
4. 거래량 동반 여부를 확인한다.
5. 방어선 이탈 시 관점을 접는다.
```

## 이 분석이 틀렸다고 보는 조건

반드시 실패 조건을 제시한다.

- 핵심 지지선 이탈
- 돌파 실패 후 거래량 동반 하락
- 스윙 최종 방어선 이탈
- 시장 지수 급락
- 관련주 동반 약세
- 수급 데이터 실패로 판단 신뢰도 하락
- 장중 데이터 신뢰도 낮음

## 거래량 강세/모멘텀 미회복

아래 조건이 동시에 나오면 거래량만으로 돌파 매수를 긍정 해석하지 않는다.

- 현재가 > 눌림목 지지가 상단
- 현재가 < 회복 확인가
- 시간가중 거래량 비율 >= 1.5
- RSI < 50
- MACD < Signal

출력:

```text
거래량은 강하지만 가격은 회복 확인가 아래이고 RSI/MACD가 아직 약합니다.
신규매수자는 회복 확인가 종가 안착 또는 눌림목 재지지 전까지 추격하지 않습니다.
```

주 전략은 `거래량 강하지만 모멘텀 미회복, 회복 확인 전 추격 금지`로 표시한다.

현재가가 20일선 아래이고 60일선 위이면 일봉 상태는 `상승` 단독 라벨 대신 `중기 상승 속 단기 조정`을 사용한다.

회복 확인가와 일봉 돌파 확인가가 같은 가격이면 `회복/돌파 공통 확인가`로 통합하고 행동 문구는 `종가 안착 + 거래량 유지 확인`으로 쓴다.

## 고거래량 음봉/역전 재돌파선 판단

단기 재돌파선은 진입 또는 회복 확인 가격이고, 목표가는 익절 후보이므로 가격 순서와 역할을 분리한다.

- 단기 재돌파선이 1차 목표와 같으면 단기 재돌파선은 별도 표시하지 않고 1차 목표/강한 저항으로만 관리한다.
- 단기 재돌파선이 2차 목표보다 높으면 단기 재돌파선에서 제거하고 `이전 고점/강한 저항`으로 재분류한다.
- 거래량 20일 평균 대비 1.5배 이상이고 종가가 시가보다 낮으면 `고거래량 음봉/매물 출회 경고`로 표시한다.
- 거래량 20일 평균 대비 2.0배 이상이고 종가가 고가 대비 크게 밀렸으면 `강한 매물 출회 경고`를 추가한다.
- 장대 음봉의 증가 거래량은 돌파 매수 근거가 아니며, 매물 소화 확인 전에는 돌파 매수 조건 충족으로 보지 않는다.

MACD 양수권/음수권은 실제 MACD 값의 부호를 따른다.

| 조건 | 설명 |
| --- | --- |
| MACD > 0 | 양수권 |
| MACD < 0 | 음수권 |
| MACD > Signal, Histogram > 0, MACD < 0 | 음수권에서 신호선 위로 회복 시도 |
| MACD > Signal, Histogram > 0, MACD > 0 | 양수권에서 모멘텀 유지 |

손익비가 매우 낮으면 최종 판단을 강하게 제한한다.

- 스윙 손익비 < 0.8 또는 회복/돌파 진입 손익비 < 0.8: `신규매수 금지에 가까움`
- 회복/돌파 진입 손익비 < 0.5: `돌파 매수 전략 성립 불가`
- 낮은 손익비 경고는 최종 문단에서 같은 의미로 반복하지 말고 한 문장으로 압축한다.

## 상태코드 기반 판단 엔진

매매 판단은 문장을 직접 조합하기 전에 `TradeState`를 먼저 만든다.

순서:

```text
데이터 계산 → 상태 분류 → 매매 상태코드 결정 → 상태코드별 문장 렌더링 → 불변조건 QA
```

최종 판단은 `final_action_state`로만 만든다.

최종 행동 상태 우선순위:

1. `DATA_INVALID`
2. `BELOW_DEFENSE`
3. `HIGH_VOLUME_BEARISH_REVERSAL`
4. `RR_STRATEGY_INVALID`
5. `RSI_OVERHEATED` + 낮은 손익비
6. `DATA_PRICE_MISMATCH`
7. 회복 확인가 아래
8. `IN_PULLBACK`
9. `BREAKOUT_INTRADAY_UNCONFIRMED`
10. `BREAKOUT_CONFIRMED`

상태별 대표 템플릿:

| final_action_state | 대표 판단 |
| --- | --- |
| `NO_BUY_HIGH_VOLUME_BEARISH` | 거래량은 급증했지만 장대 음봉으로 매물 출회 경고가 우선 |
| `NO_BUY_BAD_RR` | 가격 조건 일부가 좋아도 손익비가 맞지 않아 신규매수 부적합 |
| `NO_BUY_STRATEGY_INVALID` | 손익비가 전략 성립 기준을 밑돌아 신규매수 금지 |
| `WAIT_RECOVERY_CLOSE` | 회복 확인가 종가 안착 전 추격매수 제한 |
| `WATCH_INTRADAY_BREAKOUT` | 장중 돌파 시도이나 종가 확정 전 신규매수 보류 |
| `PARTIAL_PROFIT_NEAR_TARGET` | 목표 접근 구간으로 보유자 일부 익절 우선 |
| `DEFENSE_REQUIRED` | 방어 확인 전 매수 금지 |

상태코드가 정해진 뒤에는 보고서 렌더링 단계에서 최종 판단을 임의로 바꾸지 않는다.
